# Document Power BI Dataflows Gen1 with Copilot CoWork

In this tutorial, you'll learn how to automatically generate professional documentation for **Power BI Dataflows Gen1** using **Microsoft Copilot CoWork**.

Instead of documenting tables, columns, data types, and Power Query transformations manually, you'll use the exported **export.json** file together with a reusable CoWork Skill to create consistent documentation in Microsoft Word.

## 🧠 What this project does

This approach helps you document legacy **Power BI Dataflows Gen1** before migrating to **Dataflows Gen2** or another solution.

It allows you to:

- Export metadata from a Power BI Dataflow Gen1
- Generate comprehensive documentation automatically
- Create a reusable Microsoft Copilot CoWork Skill
- Produce consistent Word documentation for future dataflows
- Save hours of repetitive documentation work

---

## 🚀 What you'll learn

In this tutorial, you'll see how to:

- Understand why documenting Dataflows Gen1 is important before migration
- Export the **export.json** file from Power BI Service
- Generate an initial documentation template with Microsoft Copilot
- Build a reusable Copilot CoWork Skill
- Validate AI-generated documentation
- Estimate the cost of running CoWork Skills

## 🛠️ How to use

1. Export your Dataflow Gen1 as an **export.json** file.
2. Generate an initial documentation template with Microsoft Copilot.
3. Create a reusable CoWork Skill using the JSON file and your reference document.
4. Test and validate the generated documentation.
5. Reuse the skill for all future Dataflow documentation.
